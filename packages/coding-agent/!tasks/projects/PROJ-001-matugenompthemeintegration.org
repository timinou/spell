# Single Source of Truth for Theming: Matugen -> OMP

## Problem

OMP's theme (`dark.json`/`light.json`) uses hardcoded hex colors, completely disconnected from the system-wide Material 3 palette that matugen generates for every other app (ghostty, kitty, foot, gtk, hyprland, etc.).

## Solution

Add OMP as another matugen target — exactly the same pattern as ghostty, kitty, and every other app in `~/.dotfiles/wm/quickshell/dms/matugen/`. Two new files, zero OMP source changes.

OMP already supports custom themes at `~/.omp/agent/themes/{name}.json` with file-watch hot-reload. Matugen writes to that path; OMP picks it up.

### Flow After
```
Wallpaper -> Matugen -> ghostty, kitty, foot, gtk, ...
                     \-> ~/.omp/agent/themes/dankcolors.json  (NEW)
                     
OMP file watcher detects change -> hot-reloads theme
```

## Files to Create

### 1. `~/.dotfiles/wm/quickshell/dms/matugen/configs/omp.toml`

```toml
[templates.dmsomp]
input_path = 'SHELL_DIR/matugen/templates/omp.json'
output_path = 'HOME_DIR/.omp/agent/themes/dankcolors.json'
```

### 2. `~/.dotfiles/wm/quickshell/dms/matugen/templates/omp.json`

A Handlebars template outputting valid OMP theme JSON. Uses `{{colors.X.default.hex}}` (resolves to current dark/light mode, same as ghostty/foot templates).

Note: Syntax highlighting with pure M3 tokens gives only 3 hue families (primary/secondary/tertiary). This is less varied than the current VSCode Dark+ palette (7+ distinct hues). The tradeoff is cohesion with the system palette at the cost of some syntax distinguishability. If readability suffers, we can mix in dank16 ANSI colors for more variety in a follow-up.

## Verification

1. Run matugen — check `~/.omp/agent/themes/dankcolors.json` is valid JSON matching OMP schema
2. Start OMP with theme `dankcolors` — visually confirm colors match system palette
3. Change wallpaper, re-run matugen — confirm OMP hot-reloads new colors
4. Toggle dark/light via darkman — confirm matugen regenerates and OMP picks up the mode change

## Notes

- Template uses `{{X.default.hex}}` — resolves to current mode (dark or light), same as ghostty/foot templates
- OMP theme schema requires ~65 color keys; all will be mapped in the template
- The `vars` section in OMP themes allows indirection; we can use it to define M3 token aliases and reference them in `colors` for readability
- `export` section (`pageBg`, `cardBg`, `infoBg`) is optional and used for the stats dashboard — will derive from surface tokens
