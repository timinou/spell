* Problem

When plan mode activates mid-session, the entire tool set can be wiped. Every tool call returns ~"Tool X not found"~. The session becomes completely non-functional — only ~ask~ continues working (special-cased in plan-mode enforcement).

Observed in session ~14924ed3df6b6219~ (crm-dsl project): lines 1-234 had working tools, plan mode activated at line 235, and from line 239 onward every tool (bash, read, grep, find, write, org, qml, emacs_code, resolve) returned "not found".

There are two distinct bugs.

** Bug 1: ~setActiveToolsByName~ silently produces an empty tool set

** Bug 2: Org tool factory ignores plan-mode force-on

At ~tools/index.ts:196~:
#+begin_src typescript

At ~tools/index.ts:329~:

Two gates, inconsistent: ~isToolAllowed~ says yes for plan mode, but the factory independently checks ~org.enabled~ and returns ~null~ when the setting is off. The org tool is never instantiated, so it's never in ~#toolRegistry~. Plan mode promises org in its context prompt but the tool doesn't exist.

This is a secondary issue — in the observed session ALL tools were missing (Bug 1), so org would have failed regardless. But even with Bug 1 fixed, org will still fail in plan mode when ~org.enabled~ is false (e.g. projects without ~.spell/~ config).

#+begin_src typescript
async setActiveToolsByName(toolNames: string[]): Promise<void> {
    const tools: AgentTool[] = [];
    const validToolNames: string[] = [];
    for (const name of toolNames) {
        const tool = this.#toolRegistry.get(name);
        if (tool) {
            tools.push(tool);
            validToolNames.push(name);
        } else {
            logger.warn("setActiveToolsByName: tool not in registry", { name });
        }
    }
    if (tools.length === 0 && toolNames.length > 0) {
        logger.error("setActiveToolsByName: all requested tools missing from registry, keeping current set", {
            requested: toolNames,
            registrySize: this.#toolRegistry.size,
        });
        return;
    }
    this.agent.setTools(tools);
    // ... rebuild system prompt as before
}
#+end_src

** ITEM Fix 2: Org factory respects plan-mode state
:PROPERTIES:
:EFFORT: 30m
:END:

The factory runs at session init, before plan mode is enabled, so we can't fix it at factory time. Instead, when plan mode activates and needs org, create it on demand.

In ~#enterPlanMode~ at ~interactive-mode.ts~, before calling ~setActiveToolsByName~:

1. Check if "org" is missing from the tool registry
2. If so, instantiate OrgTool and register it

This requires either:
- (a) Exposing a ~registerTool(name, tool)~ method on AgentSession, OR
- (b) Having ~#enterPlanMode~ call into the tool creation infrastructure

Option (a) is simpler. Add to ~AgentSession~:
#+begin_src typescript
registerTool(name: string, tool: AgentTool): void {
    this.#toolRegistry.set(name, tool);
}
#+end_src

Then in ~#enterPlanMode~:
#+begin_src typescript
// Ensure org tool exists for plan mode (factory skips it when org.enabled is false)
if (!this.session.getToolByName("org")) {
    const orgTool = new OrgTool(this.session as ToolSession);
    this.session.registerTool("org", orgTool);
}
#+end_src
