* Problem

=mapSessionEvent= in =qml-mode.ts= uses module-level mutable state (=currentMessageId=, =messageCounter=, =lastTextLength=). This makes it:
1. Untestable in isolation — tests share state, ordering matters
2. Broken if multiple QML sessions ran concurrently (unlikely today, but latent bug)

* Approach

Extract into a =SessionEventMapper= class with instance state. Export from a new file so tests import it directly without pulling in QML bridge dependencies.

:END:

** ITEM Write unit tests
:PROPERTIES:
:EFFORT: 15m
:END:

Create =packages/coding-agent/test/modes/qml-event-mapper.test.ts=.

Test file uses =bun:test= (=describe=, =it=, =expect=, =beforeEach=). Each test gets a fresh =new SessionEventMapper()=.

Contracts to test (each is one =it= block):

*** Message ID threading
- =message_start= returns an object with a non-empty =id=
- =message_update= and =message_end= return the same =id= as the preceding =message_start=

*** Delta text accumulation
- First =message_update= with ="hello"= → =text: "hello"=
- Second =message_update= with ="hello world"= → =text: " world"= (delta only)

*** Delta reset on new message
- After a full start/update/end cycle, a new =message_start= resets the cursor
- First update of the new message returns full text, not delta from previous message

*** Suppression of empty deltas
- =message_update= with same full text as last time → returns =null=

*** Tool field mapping
- =tool_execution_start= maps =toolCallId= → =id=, =toolName= → =name=, =intent= → =details=
- =tool_execution_end= maps =toolCallId= → =id=, =toolName= → =name=, =isError= preserved

*** Agent busy events
- =agent_start= → ={ type: "agent_busy", busy: true }=
- =agent_end= → ={ type: "agent_busy", busy: false }=

*** Unrecognized events return null
- =auto_compaction_start= or any non-UI event → =null=

Test helpers (defined in the test file):
#+begin_src typescript
function assistantMsg(content: Array<{ type: string; text?: string }> = []) {
  return { role: "assistant" as const, content };
}

function textBlock(text: string) {
  return { type: "text" as const, text };
}
#+end_src

These construct minimal =AgentMessage=-shaped objects. The mapper uses duck typing (=hasContent=) so full =AgentMessage= instances aren't needed.
