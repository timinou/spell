* Initial Message

tested?

* Problem

=spell --qml= has no automated tests for the QML UI layer. The recent ChatPanel fixes (ghost bubble filtering, tool result display, user_message handling) have no verification beyond =bun check:ts=. The QML bridge already supports bidirectional messaging and screenshots, but there's no headless test infrastructure to leverage these capabilities from =bun:test=.

* Architecture

Two-layer test approach:

1. *Pure TS unit tests* for =SessionEventMapper= (extend existing =qml-event-mapper.test.ts= with new filter behavior tests)
2. *Integration tests* using the real C++ bridge binary (=spell-qml-bridge=) in offscreen mode, exercising the full JS→bridge→QML→bridge→JS event loop

#+begin_example
bun:test
  └── QmlTestHarness (packages/qml/src/test-harness.ts)
        ├── wraps QmlProcess in stdio mode (NOT daemon — avoids socket collisions)
        ├── injects QT_QPA_PLATFORM=offscreen via env option
        ├── provides: sendMessage(), query(), screenshot(), reset(), teardown()
        └── communicates via bridge JSON protocol over stdin/stdout
  └── ChatPanelTestHarness.qml (co-located in panels/ directory)
        ├── ApplicationWindow root (required for grabWindow() screenshots)
        ├── loads ChatPanel as child, wires bridge.messageReceived → handleMessage
        ├── handles "query" messages for model introspection
        └── handles "reset" messages to clear model between tests
#+end_example

* Key Design Decisions

** stdio mode, not daemon mode

Daemon mode uses a singleton Unix socket path (=$XDG_RUNTIME_DIR/spell-qml-bridge.sock=). Concurrent test files would collide. Stdio mode gives each test file its own isolated process with stdin/stdout pipes.

** Test wrapper co-located in panels/ directory

ChatPanel.qml imports =".." as SpellUI= — relative import. The test wrapper must be in the same directory (or a subdirectory) for Qt's QML import resolver to find SpellUI. Place =ChatPanelTestHarness.qml= in =packages/coding-agent/src/modes/qml/panels/=.

** ApplicationWindow root for screenshots

=windowmanager.cpp= line 223: =screenshotWindow= does =qobject_cast<QQuickWindow*>(rootObjects().first())=. A plain Item root fails. The test wrapper must use =ApplicationWindow= (or =Window=).

** One bridge per test file, reset between tests

Spawning a Qt process takes ~500ms. Per-file (=beforeAll/afterAll=) amortizes the cost. Between tests, send a "reset" message to clear the ListModel. The =QmlTestHarness.reset()= sends ={type:'reset'}=, wrapper calls =messagesModel.clear()= and resets properties.

** Graceful skip when binary is absent

Not all machines have Qt6 dev headers. Tests check =isBridgeAvailable()= in =beforeAll= and skip the suite with a warning if the binary isn't built. This prevents CI failures on machines without Qt.

** Structural assertions over screenshot assertions for CI

Offscreen rendering uses Qt's software rasterizer — font metrics differ across distros. Use model introspection (message count, text content, role, isStreaming) for correctness assertions. Screenshots are a development aid, not CI gates. If used, set pixelmatch threshold ≥5%.

High-level test helper that wraps =QmlProcess= in stdio mode with offscreen rendering.

API:

#+begin_src typescript
export interface QmlTestHarnessOptions {
    /** Additional env vars beyond QT_QPA_PLATFORM=offscreen */
    env?: Record<string, string>;
    width?: number;
    height?: number;
}

export class QmlTestHarness {
    /** Spawn bridge in stdio mode, load the given QML file. */
    async setup(qmlPath: string, props?: Record<string, unknown>): Promise<void>;

    /** Send a message to the QML window. */
    async sendMessage(payload: Record<string, unknown>): Promise<void>;

    /**
     * Query QML state via introspection protocol.
     * Sends {type:'query', query:queryName}, waits for
     * {type:'query_response', query:queryName, result:...} event.
     * The test wrapper QML must handle "query" messages.
     */
    async query<T = unknown>(queryName: string): Promise<T>;

    /** Capture screenshot, return the file path. */
    async screenshot(savePath?: string): Promise<string>;

    /** Reset QML state between tests. Sends {type:'reset'}. */
    async reset(): Promise<void>;

    /** Kill bridge process. */
    async teardown(): Promise<void>;
}
#+end_src

Implementation details:
- Uses =QmlProcess.spawnStdio()= (not =ensure()= which tries daemon first)
- Injects =QT_QPA_PLATFORM=offscreen= into env
- Sends a =load= command after spawn (same as =QmlBridge.launch()=)
- =query()= sends message, then waits for matching event using =QmlProcess.waitFor()= with a 5s timeout
- =screenshot()= sends screenshot command, waits for screenshot event, returns path
- Window ID is fixed to ="test"= for simplicity

Also export =isBridgeAvailable= from the package so test files can check.

#+begin_src qml
import QtQuick 2.15
import QtQuick.Controls 2.15
import ".." as SpellUI

ApplicationWindow {
    visible: true
    width: windowWidth || 800
    height: windowHeight || 600
    title: windowTitle || "Test"
    color: SpellUI.SpellTheme.background

    ChatPanel {
        id: chatPanel
        anchors.fill: parent
    }

    Connections {
        target: bridge
        function onMessageReceived(payload) {
            if (payload.type === "query") {
                handleQuery(payload)
            } else if (payload.type === "reset") {
                handleReset()
            } else {
                chatPanel.handleMessage(payload)
            }
        }
    }

    function handleQuery(payload) {
        var queryName = payload.query
        var result = null

        if (queryName === "messages") {
            // Serialize the full ListModel as an array
            var msgs = []
            for (var i = 0; i < chatPanel.messagesModel.count; i++) {
                msgs.push(chatPanel.messagesModel.get(i))
            }
            result = msgs
        } else if (queryName === "messageCount") {
            result = chatPanel.messagesModel.count
        } else if (queryName === "isStreaming") {
            result = chatPanel.isStreaming
        } else if (queryName === "lastMessage") {
            var count = chatPanel.messagesModel.count
            result = count > 0 ? chatPanel.messagesModel.get(count - 1) : null
        }

        bridge.send({
            type: "query_response",
            query: queryName,
            result: result
        })
    }

    function handleReset() {
        chatPanel.messagesModel.clear()
        chatPanel.isStreaming = false
        chatPanel.tokenCount = 0
        chatPanel.modelName = ""
        bridge.send({ type: "reset_done" })
    }
}
#+end_src

Contracts to verify:

*** Ghost bubble filtering

Send =message_start= with role "user" → query messageCount → expect 0 (mapper blocks it, but this test sends directly to QML, so we test the ChatPanel handler). Actually, the mapper test covers the filtering. The integration test should verify the end-to-end: if a =message_start= arrives with role "user" (bypassing mapper), ChatPanel appends it with role "user" not "assistant" (using =msg.role || "assistant"=). Verify via model query that the role field is "user".

*** Assistant message streaming lifecycle

1. Send =message_start {id:"msg-1", role:"assistant"}=
2. Query: messageCount === 1, last message role === "assistant", isStreaming === true, text === ""
3. Send =message_update {id:"msg-1", text:"Hello"}=
4. Query: last message text === "Hello"
5. Send =message_update {id:"msg-1", text:" world"}=
6. Query: last message text === "Hello world"
7. Send =message_end {id:"msg-1"}=
8. Query: last message isStreaming === false, chatPanel.isStreaming === false

*** Tool execution lifecycle

1. Send =tool_start {id:"t-1", name:"bash", details:"Running build"}=
2. Query: messageCount === 1, last message role === "tool", name === "bash", text === "Running build", isStreaming === true
3. Send =tool_end {id:"t-1", name:"bash", isError:false, details:"Build succeeded"}=
4. Query: last message isStreaming === false, text === "Build succeeded"

*** user_message handling

1. Send ={type:"user_message", text:"hello world"}=
2. Query: messageCount === 1, last message role === "user", text === "hello world"

*** agent_busy state

1. Send =agent_busy {busy:true}= → query isStreaming === true
2. Send =agent_busy {busy:false}= → query isStreaming === false

*** Screenshot smoke test (optional, platform-dependent)

1. Send a sequence of messages (user, assistant, tool)
2. Take screenshot → verify file exists and is a valid PNG (check magic bytes)
3. Do NOT do pixel comparison in CI — just verify the screenshot mechanism works
