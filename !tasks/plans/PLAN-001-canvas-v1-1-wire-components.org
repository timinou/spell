* Problem

Canvas v1 shipped the full infrastructure — Canvas.qml, ContentBlock.qml, DataTable/DiffView/TreeView components, message protocol, skill, tests — but has three functional gaps that prevent daily use:

1. *ContentBlock doesn't route to components.* The switch in ContentBlock.qml only handles =markdown= and =image=. The =table=, =diff=, =tree= types fall through to a JSON dump fallback. The actual component QML files exist and are tested, but the integration layer skips them.

2. *SKILL.md documents wrong data shapes.* The skill examples show flat arrays for table columns (e.g. =["Col A", "Col B"]=) when DataTable.qml expects =[{key, label, width?}]= objects. Diff shows ={left, right}= when DiffView expects ={hunks: [{header, lines}]}=. Tree shows a bare node when TreeView expects ={nodes: [...]}.= An agent following the docs will generate data the components silently render as empty.

3. *No reusable launcher.* Canvas.qml is an Item, not a window. Every session the agent must write a ~25-line ApplicationWindow wrapper — identical boilerplate every time. The test harness IS this wrapper plus test-only query/reset handlers.

Secondary gaps (can ship after):
4. Component events (rowClicked, lineClicked, nodeClicked) don't propagate to the bridge — interactive features silently discarded
5. No =update= action to modify a single block — must re-send all via =set=
6. Canvas.qml name shadows QtQuick's built-in Canvas type

** Phase 2: Event propagation

*** ITEM Propagate component events to bridge
:PROPERTIES:
:EFFORT: 1.5h
:END:

1. Add a generic signal to ContentBlock: =signal componentEvent(string eventType, var eventData)=

2. When wiring components in ContentBlock Loader, connect their signals:
   - =DataTable.onRowClicked: root.componentEvent("row_click", {blockId: root.blockId, rowIndex: idx, rowData: data})=
   - =DiffView.onLineClicked: root.componentEvent("line_click", {blockId: root.blockId, ...})=
   - =TreeView.onNodeClicked: root.componentEvent("node_click", {blockId: root.blockId, ...})=

   Problem: signals from Loader items can't be connected declaratively (the items don't exist until the Loader loads them). Use =Connections { target: contentLoader.item }= with guards.

3. In Canvas.qml, connect each ContentBlock delegate's =componentEvent= signal to =bridge.send({action: "event", ...})=.

} else if (action === "update") {
