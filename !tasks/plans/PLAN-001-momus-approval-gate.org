* Problem

Momus (plan quality reviewer) is currently prompt-driven: the =plan-mode-active.md= template tells the agent to ask the user if they want Momus review. This is fragile — the agent may skip the ask or phrase it poorly. It should be a code-driven option in the plan approval UI.

* Approach

When =exit_plan_mode= fires and the approval selector appears, add a 4th option "Review with Momus" (only when =planModeUltraplan= is true). Selecting it re-enters plan mode and injects a synthetic message instructing the agent to spawn Momus and act on results. After revision, the agent calls =exit_plan_mode= again, re-triggering the selector — creating a natural review loop.
